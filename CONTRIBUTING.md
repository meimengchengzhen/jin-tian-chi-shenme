# 关于贡献

感谢你愿意花时间看到这里 🍳。

**本项目当前阶段不接受直接的代码 / 内容 / 数据 / 设计贡献。**

「饭搭子 · 今天吃什么」是作者本人维护的个人作品，目前处于持续打磨阶段，所有迭代节奏、
内容口味、UI 取舍都由作者自己把控。为了保持作品的整体一致性与著作权清晰，本仓库
**没有开放公开协作**，也没有采用开源许可证 —— 详见 [LICENSE](./LICENSE)。

如果你有想法，仍然非常欢迎，但请按以下方式与作者沟通：

## 一、可以做什么

- **提 Issue 反馈 Bug 或体验建议**
  在 [GitHub Issues](../../issues/new) 留言，描述你遇到的问题或想到的功能。
  作者会自行评估是否在后续版本中实现，但**不保证回复或采纳**。

- **联系作者**
  如果你有合作、引用、转载、教学或其他超出 LICENSE 中「有限允许」范围的需求，
  请通过本仓库主页公开的联系方式与作者联系，**获得书面同意后**再行动。

## 二、不接受什么

- **直接提交 Pull Request 修改源码、菜谱、文案、设计稿或数据**
  即便是「只是改个错别字」「只是加一道菜」，也请先开 Issue 与作者确认后再行动。
  未经事先同意提交的 PR，作者**无义务审阅、无义务解释拒绝原因，且不一定回复**。

- **批量抓取、镜像、转载、二次发布本仓库或部署网页的内容**
  详见 [LICENSE](./LICENSE)。

- **将本项目（或其任意片段）用于商业产品、付费课程、AI 训练数据**
  同样需要单独获得书面授权。

## 三、关于已经提交的 PR / Issue

- 任何形式的提交（Issue、Pull Request、邮件附件、聊天消息等）一旦发出，
  即视为你**主动放弃**对该提交内容的独占权利，并允许作者在后续版本中
  自由参考、改写或不采用，且**不构成必须合并、署名或回报的义务**。
- 是否合并、是否采用、是否署名，**完全由作者决定**。
- Pull Request 的存在**不保证被接受**；提交后被关闭、长期搁置或在重构中丢失，
  均属正常情况，请勿据此追责。

## 四、未来是否会开放

不排除。如果项目进入更稳定的阶段，作者可能会：

- 选择性开放某些数据目录（例如菜谱条目）的协作；
- 重新评估并更换为更明确的许可证；
- 发布单独的「贡献者协议（CLA）」。

任何调整都会在本文件与 [LICENSE](./LICENSE) 中同步更新，以彼时版本为准。
